This is a list of tags used in the HTML language. Each tag starts with a tag opener (a less than sign) and ends with a tag closer (a greater than sign). Many tags have corresponding closing tags which are identical except for a slash after the tag opener.

Some tags take parameters, called attributes. The names of tags and attributes are not case sensitive.

Title

The title of a document is given between title tags. There should only be one title in any node. It should identify the content of the node in a fairly wide context, and should ideally fit on one line. The title may be used to identify the node in a history list or to label the window displaying the node.

Next ID

Obsolete: NeXT Browser only. May be ignored. This tag takes a single attribute which is the number of the next document-wide numeric identifier to be allocated. Human writers of HTML usually use mnemonic alpha identifiers.

Base Address

Anchors specify addresses of other documents relative to the address of the current document. If a document is mailed or visible with more than one address, the browser needs the base address to correctly deduce external document addresses.

The format of this tag is not yet specified. NOT CURRENTLY USED.

Anchors

The text between the opening and closing tag is either the start or destination (or both) of a link. Attributes of the anchor tag are as follows.

HREF
If the HREF attribute is present, the anchor is sensitive text: the start of a link. The reader is presented with another document whose address is defined by the value of the HREF attribute. This allows for HREF=#identifier to refer to another anchor in the same document.
NAME
The attribute NAME allows the anchor to be the destination of a link. The value is that part of a hypertext address which follows the hash sign.
TYPE
An attribute TYPE may give the relationship described by the hypertext link. The type is expressed by a string for extensibility. The default relationship if none other is given is void.

All attributes are optional, although one of NAME and HREF is necessary for the anchor to be useful.

IsIndex

This tag informs the reader that the document is an index document. The node may be queried with a keyword search by suffixing the node address with a question mark, followed by keywords separated by plus signs.

Plaintext

This tag indicates that all following text is to be taken literally, up to the end of the file. Plain text is represented with fixed width characters and significant line breaks. There is no closing tag.

Example sections

These styles allow fixed-width characters to be embedded absolutely as is into the document. Between the opening and closing tags:

  • The text may contain any ISO Latin printable characters, so long as it does not contain the closing tag in full.
  • Line boundaries are significant, interpreted as a move to the start of a new line.
  • The ASCII Horizontal Tab character should be interpreted as the smallest positive nonzero number of spaces leaving the character count as a multiple of 8. Its use is not recommended.

The LISTING tag is portrayed so that at least 132 characters fit on a line. The XMP tag is portrayed so that at least 80 characters fit on a line.

Paragraph

This tag indicates a new paragraph. The exact representation (indentation, leading, etc.) is not defined here, and may be a function of other tags or style sheets.

Headings

Several levels (at least six) of heading are supported. H1 is the highest level and is recommended for the start of a hypertext node. A hypertext document tends to need fewer levels of heading than a normal document.

Address

This tag is for address information, signatures, etc., normally at the top or bottom of a document. Typically it is italic and/or right justified or indented.

Highlighting

The highlighted phrase tags may occur in normal text and may be nested. For each opening tag there must follow a corresponding closing tag. NOT CURRENTLY USED.

Glossaries

A glossary (or definition list) is a list of paragraphs each of which has a short title alongside it. This format is also useful for presenting a set of named elements to the reader.

Lists

A list is a sequence of paragraphs, each preceded by a special mark or sequence number. The opening list tag must be immediately followed by the first list element. Opening list tags are:

UL
A list of multi-line paragraphs, typically separated by some white space.
MENU
A list of smaller paragraphs. Typically one line per item, more compact than UL.
DIR
A list of short elements, less than one line. Typical style is to arrange in four columns.

The closing tag must obviously match the opening tag.
